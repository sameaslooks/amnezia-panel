## 📌 О проекте

**Amnezia VPN Panel** — это профессиональная веб-панель для управления VPN-серверами на базе протокола AmneziaWG2. Проект предоставляет полный цикл управления: от установки сервера до мониторинга трафика и лимитов пользователей.

### 🎯 Для кого это?
- **Провайдеры VPN-услуг** — полный контроль над пользователями и серверами
- **Администраторы** — централизованное управление несколькими серверами
- **Разработчики** — открытый код для кастомизации и интеграций

### 🏢 Коммерческое использование
Проект идеально подходит для построения VPN-бизнеса. Для коммерческих целей доступен расширенный **биллинг-модуль** (закрытый), который добавляет:
- Интеграцию с платежными системами
- Автоматическое создание/продление подписок
- Тарифные планы и промокоды
- Email-уведомления
- API для внешних биллинговых систем

*По вопросам приобретения биллинг-модуля обращаться к разработчику*

## ✨ Ключевые возможности

### 🖥️ Управление серверами
- Добавление неограниченного количества серверов через SSH
- Автоматическая установка AmneziaWG на удалённый сервер
- Мониторинг статуса (онлайн/офлайн, работа контейнера)
- Управление контейнером (запуск, остановка, перезапуск)
- Поддержка аутентификации: пароль, приватный ключ, ключ + sudo
- Потоковая передача логов установки через WebSocket

### 👥 Управление пользователями
- Роли: администратор / обычный пользователь
- Лимиты трафика и срок действия подписки
- Автоматическая блокировка при превышении лимитов, разблокировка - при повышении лимита трафика или продлении срока. 
- Ограничение количества клиентов на пользователя
- Возможность создавать/удалять собственных VPN-клиентов для обычных пользователей

### 📱 VPN-клиенты
- Генерация ключей и конфигураций для каждого устройства
- Статистика трафика и время последнего handshake
- AmneziaWG QR-коды и AmneziaVPN-ссылки для быстрого импорта
- Принудительная синхронизация клиентов между серверами и базой данных, если мигрируете уже настроенный сервер AmneziaWG в панель. 

### 📊 Мониторинг и статистика
- Дашборд с общей статистикой (серверы, клиенты, трафик)
- Графики трафика за последние 30 дней
- Топ пользователей по потреблённому трафику
- Автоматическое выявление проблемных серверов (на момент - отключённых) и истекающих подписок
- Периодический сбор трафика с защитой от сброса счётчиков
- Интеграция с Prometheus (графики загрузки CPU, RAM, максимального Bandwith Download/Upload при прогоне на автоматически выбранном сервере Speedtest)

### 🔧 Технические особенности
- Полная поддержка параметров обфускации AmneziaWG (Jc, Jmin, Jmax, S1-S4, H1-H4, I1-I5)
- Автоматическая блокировка/разблокировка на уровне маршрутов.
- Хранение приватных ключей клиентов в БД для восстановления
- Работа через Docker-контейнер

### 💼 Биллинг-модуль (закрытый)
Для коммерческого использования доступен расширенный биллинг-модуль, добавляющий:
- Полностью автоматизированного бота, готового к работе с пользователями и администраторами, имеющего всю функциональность панели. 
- Интеграцию с крупными платёжными системами
- Тарифные планы и автопродление подписок
- Систему обращений в поддержку
- Историю платежей и возможность получения финансовых отчётов

*По вопросам приобретения биллинг-модуля обращаться к разработчику.*

## ⚡ Быстрый старт

### Системные требования
- **ОС**: Linux (рекомендуется Ubuntu 20.04 / 22.04 / 24.04)
- **Процессор**: 1 vCPU (минимум)
- **RAM**: 1 GB (минимум), 2 GB (рекомендуется)
- **Диск**: 10 GB свободного места
- **Docker**: версия 20.10+
- **Docker Compose**: версия 2.0+

### Установка

```bash
# Клонирование репозитория
git clone https://github.com/sameaslooks/amnezia-panel
cd amnezia-panel

# Настройка окружения
cp .env.example .env
cp docker-compose.yml.example docker-compose.yml
cp nginx.conf.example nginx.conf

# Редактирование конфигурации
nano .env  # Обязательно укажите JWT_SECRET и смените пароль БД!
nano docker-compose.yml   # Обязательно смените пароль БД PostgresDB! Укажите такой же, какой и в .env.
nano nginx.conf   # Если планируете пользоваться HTTP-подключением заместо HTTPS - обязательно выполните настройку в данном файле! Для HTTPS вносить правки не требуется. 

# Запуск панели
docker compose up -d

# Просмотр логов
docker compose logs -f

```
- Открыть панель возможно по адресу `http://SERVER-IP` или `https://SERVER-IP`
- Для авторизации, стандартные логин и пароль: admin:admin
- **Внимание!** Не рекомендуется использовать локальное управление сервером (именуется "local" в панели, изначально отключён), поскольку в случае миграции сервера данный способ подключения **может сломать сломать Ваши настройки**! Также, локальное взаимодействие более *не дорабатывается*, и даже для установки AmneziaWG-сервера на той же машине, где стоит и панель, **настоятельно рекомендую создавать подключение по SSH**! (меню "Servers")

## 🔧 Конфигурация
Переменные окружения (.env)
```env

# Обязательные параметры
JWT_SECRET=your-supert-secret-long-password
DATABASE_URL=postgresql://amnezia:YOUR-LONG-BIG-PASSWORD@postgres:5432/apanel_db

# Опциональные параметры
FRONTEND_DIR=/frontend
BACKEND_DIR=/app
DEBUG=false
```

Расширенная настройка

  * SSL-сертификаты: если используете Nginx и letsencrypt - настройте домен и поддомен панели в .env, если используете сертификаты иного провайдера - настройте пути в nginx.conf
  * Резервное копирование: вся база данных находится в ./postgres/data. Регулярно резервируйте данную папку.
